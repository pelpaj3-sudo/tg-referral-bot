# Реферальний Telegram-бот

Бот нараховує користувачам бонус за кожного друга, який приєднався за реферальним
посиланням і підписався на канал. Накопичені кошти можна вивести після досягнення
мінімальної суми — заявку обробляє адміністратор вручну прямо в чаті.

## Можливості

- Реферальне посилання виду `https://t.me/BotName?start=<id>`
- Нарахування бонусу лише після підписки на канал (верифікація через `getChatMember`)
- Баланс, список рефералів зі статусами ✅/⏳, виведення коштів
- Заявки на виведення з кнопками «Виплачено» / «Відхилити» для адміна
- Адмін-панель прямо в боті: статистика, список заявок, розсилка всім користувачам
- Дані зберігаються у файлі `data/db.json` (без зовнішньої БД)

## Налаштування

1. Встановіть залежності:
   ```bash
   npm install
   ```
2. Скопіюйте `.env.example` у `.env` і заповніть:
   - `BOT_TOKEN` — токен від [@BotFather](https://t.me/BotFather)
   - `ADMIN_IDS` — ваш Telegram ID (і колег через кому). Дізнатись ID можна у [@userinfobot](https://t.me/userinfobot)
   - `CHANNEL_USERNAME` — публічний канал, напр. `@my_channel`. Для приватного каналу
     залиште це поле порожнім і заповніть `CHANNEL_ID` (chat id виду `-100...`) та
     `CHANNEL_INVITE_LINK`
   - `REFERRAL_BONUS` — сума за реферала (за замовчуванням 7)
   - `MIN_WITHDRAWAL` — мінімальна сума виведення (за замовчуванням 70)
3. **Важливо:** додайте бота адміністратором каналу — інакше перевірка підписки
   (`getChatMember`) не працюватиме.
4. Поки канал ще не створено, можна лишити `CHANNEL_USERNAME`/`CHANNEL_ID` порожніми —
   бот тимчасово вважатиме всіх користувачів верифікованими, щоб можна було
   протестувати решту функціоналу. Після створення каналу додайте його в `.env`,
   перезапустіть бота — і верифікація стане обов'язковою.
5. Запуск:
   ```bash
   npm start
   ```

## Як це працює для користувача

1. Переходить за реферальним посиланням → `/start <id_запросившого>`
2. Бот просить підписатися на канал і натиснути «Я підписався»
3. Якщо підписка підтверджена — користувача верифіковано, а тому, хто його запросив,
   нараховується бонус
4. У меню: баланс, своє посилання, список рефералів, виведення коштів
5. При виведенні бот питає реквізити (картка/гаманець) і створює заявку — вона одразу
   надходить усім адмінам з кнопками підтвердження

## Адмін-панель

Кнопка «🛠 Адмін-панель» (або команда `/admin`) доступна лише ID з `ADMIN_IDS`:

- **📊 Статистика** — кількість користувачів, верифікованих, зарахованих рефералів,
  нараховано/виплачено, заявки в очікуванні, топ-10 реферерів
- **📋 Заявки на виведення** — список активних заявок з кнопками «Виплачено» / «Відхилити»
  (при відхиленні кошти повертаються користувачу на баланс)
- **📢 Розсилка** — надішліть текст, і бот розішле його всім користувачам бота

## Структура проєкту

```
src/
  config.js     — читання .env
  db.js         — зберігання даних (users, withdrawals) у data/db.json
  texts.js      — усі тексти повідомлень бота
  keyboards.js  — клавіатури (reply/inline)
  bot.js        — вся логіка бота
data/
  db.json       — створюється автоматично при першому запуску
```

## Нотатки з дизайну рішення

- Виведення завжди на всю суму балансу (без ручного вводу суми) — простіше й менше
  кроків для користувача.
- Заявка на виведення одразу списує суму з балансу; якщо адмін відхилить — кошти
  повертаються.
- Дані зберігаються у звичайному JSON-файлі — для обсягів у сотні/тисячі користувачів
  цього достатньо. За потреби масштабування можна перейти на SQLite/PostgreSQL,
  зберігши той самий інтерфейс `db.js`.
